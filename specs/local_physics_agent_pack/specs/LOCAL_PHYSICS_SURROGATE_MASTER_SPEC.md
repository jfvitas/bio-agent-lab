# LOCAL PHYSICS SURROGATE MASTER SPEC
Version: 1.0
Status: Authoritative
Applies to: Architect, implementation, reviewer, QA, and user-simulation testing agents

## 1. Objective

Build a two-stage system:

1. A **local physics target generation pipeline** that uses higher-fidelity external calculations on representative local site environments.
2. A **surrogate property model** that predicts refined per-site physical descriptors from local topology + 3D environment.
3. A **graph enrichment pipeline** that uses the surrogate outputs as node features in downstream binding models.

This subsystem must support protein-ligand, protein-peptide, and protein-protein prediction workflows.

## 2. Key principle

The project must not try to run expensive QM/MM on all atoms in all proteins during feature generation or prediction.
Instead it must:

- extract site environments from many structures
- cluster them into archetypes
- run expensive analysis only on archetypes
- train a surrogate model on those labels
- apply the surrogate to all sites in the full corpus

## 3. Mandatory outputs

The subsystem must eventually produce these artifact classes:

- `artifacts/site_envs/<run_id>/*.parquet`
- `artifacts/archetypes/<run_id>/*.parquet`
- `artifacts/external_analysis/<analysis_batch_id>/*`
- `artifacts/physics_targets/<analysis_batch_id>/*.parquet`
- `artifacts/surrogate_training/<run_id>/*`
- `artifacts/site_physics/<run_id>/*.parquet`
- `artifacts/graphs/<run_id>/*`

## 4. Descriptor targets to predict

Version 1 target vector for each site must contain exactly these fields:

1. refined_partial_charge
2. electrostatic_potential
3. electric_field_magnitude
4. donor_strength
5. acceptor_strength
6. polarizability_proxy
7. effective_steric_radius
8. desolvation_penalty_proxy
9. protonation_preference_score
10. metal_binding_propensity
11. aromatic_interaction_propensity
12. local_environment_strain_score

Optional future fields must not be added until versioned:
- dipole tensor components
- quadrupole descriptors
- cation_pi_propensity
- explicit halogen bond propensity
- water-mediated interaction propensity

## 5. Inputs to the surrogate model

The surrogate model input must combine:

### A. Site identity / chemistry scaffold
- motif_class
- atomic_number where applicable
- formal_charge
- aromatic_flag
- ring_membership
- hybridization_proxy
- backbone_vs_sidechain flag if protein
- metal flag if relevant

### B. Local topology
- covalent neighbors
- bond orders where available
- graph distances

### C. Local 3D environment
- neighborhood graph within 8 Å
- shell-based environment descriptors
- coordinates or equivariant geometry

### D. Structure context
- SASA
- burial score
- residue depth
- interface flag
- normalized B-factor
- occupancy
- secondary structure class
- optional apo/holo state tag

## 6. Model architecture requirements

The surrogate model must be geometry-aware.

Preferred architecture:
- local equivariant GNN

Acceptable alternatives:
- message-passing GNN + explicit geometric features
- EGNN / SE(3)-equivariant style model

Forbidden:
- lookup-table-only implementation
- model that ignores 3D geometry entirely
- model that uses only atom labels without local environment

## 7. Training data construction requirements

The training data must come from:
- real local site environments extracted from structures
- representative archetypes selected by motif class
- higher-fidelity physics outputs generated by offline analysis

Training data must not be created by arbitrary synthetic enumeration of residue combinations alone.

## 8. Where external analysis fits

External analysis is outside the ordinary day-to-day feature pipeline.

External analysis is performed separately on:
- representative motif archetypes
- curated local fragments
- selected probe interactions

The outputs are then ingested back into the project as supervised labels.

## 9. What must be built in-project

The project code must implement all of the following:

- structure parsing
- site identification
- environment vector generation
- archetype clustering
- fragment export generation
- external analysis job manifest generation
- result ingestion
- target table assembly
- surrogate model training
- surrogate inference
- graph enrichment
- QA and acceptance tests

## 10. What is explicitly outside the normal in-project runtime

The following work is performed as separate analysis batches outside normal feature generation:
- ORCA / QM local calculations
- APBS electrostatic map generation
- OpenMM local MM energy jobs
- manual or semi-manual curation of failed archetypes

These jobs must produce machine-readable output manifests so the project can ingest them reproducibly.

## 11. Acceptance rules

The subsystem is not complete until:
- site environments are extracted deterministically
- archetype clustering exists and is versioned
- external analysis queues are generated automatically
- result ingestion is automated
- the surrogate model is trainable from ingested targets
- surrogate outputs are attached to graph nodes
- the enriched graphs are usable by downstream training code
