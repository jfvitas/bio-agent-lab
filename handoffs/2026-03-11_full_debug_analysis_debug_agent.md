## 2. Confirmed Bugs

### BUG-1: Fallback pair key incompatible with parser (SEVERITY: HIGH)

**File:** `assay_merge.py:105-110` → consumed by `pairing.py:42`

The fallback branch of `pair_identity_key()` produces a key with only 4 pipe-separated parts:
```python
# assay_merge.py ~line 105-110
return "|".join([pdb_id, chain_str, "none", task_type])
# Produces: "1ABC|A|none|protein_ligand" (4 parts)
```

But `parse_pair_identity_key()` expects exactly 5 parts and returns `None` otherwise:
```python
# pairing.py ~line 42
parts = key.split("|")
if len(parts) != 5:
    return None
```

**Impact:** Any downstream code using `parse_pair_identity_key()` on a fallback-generated key gets `None`, breaking ligand matching, deduplication, and training example assembly for entries that hit this fallback.

### BUG-2: Metal ion misclassification in disambiguate_roles (SEVERITY: MEDIUM)

**File:** `rcsb_classify.py:596`

When `disambiguate_roles()` detects that any small molecule exists in the entry, it marks **all** metal ions as `"metal_mediated_contact"` regardless of whether each individual metal is actually mediating a small-molecule contact.

**Impact:** Entries with both catalytic metals and unrelated small-molecule ligands will have their catalytic metals misclassified, affecting entity counts and downstream classification.

### BUG-3: BindingDB affinity unit inconsistency (SEVERITY: MEDIUM)

**File:** `bindingdb.py` — `_parse_affinity()`

The function computes `log10 = round(math.log10(std), 6)` where `std` is in **nanomolar** (nM). This produces `log10(nM)` values. However, standard biochemical convention and other sources in this pipeline (SKEMPI, PDBbind) typically use `-log10(M)` (pKd/pKi). The field name `assay_value_log10` does not disambiguate.

**Impact:** Mixed affinity scales in merged assay data. A Kd of 10 nM would be stored as `log10(10) = 1.0` from BindingDB but as `-log10(10e-9) = 8.0` from other sources.

### BUG-4: `_apply_mmcif_supplement` mutates shared dicts (SEVERITY: LOW-MEDIUM)

**File:** `rcsb_classify.py`

`_apply_mmcif_supplement()` mutates the original entity dictionaries from the GraphQL response in place. If the same raw data dict is reused or inspected later, it will contain injected mmCIF fields.

**Impact:** Subtle data corruption under caching or retry. Currently low severity because pipeline is single-pass.

## 5. Data Integrity Risks

### RISK-1: Affinity Scale Mixing (HIGH)
BindingDB stores `log10(nM)` while other conventions use `-log10(M)`. The `assay_merge.py` conflict detection uses a log10-spread threshold of 0.3/1.0, but this assumes a common scale. Mixed scales make the conflict detector unreliable.

### RISK-2: Pair Key Collision
`pair_identity_key()` sorts chains and uses `|`-separated concatenation. Two biologically distinct complexes that share PDB ID, chain labels, and ligand identifiers would collide.

### RISK-3: Train/Test Leakage via Graph Connectors
`merge_external_sources()` adds STRING/Reactome/BioGRID edges to the graph **before** dataset splitting. If graph features are used as ML features, information about test-set proteins leaks through shared graph neighbors.

### RISK-4: Stale ESM Cache
`_esm_embedding()` caches by MD5 of sequence. No model-version tag. If ESM model version changes, stale embeddings silently persist.

### RISK-5: Non-Deterministic Provenance Timestamps
`_edge_provenance()` and `_utc_now()` use `datetime.now(timezone.utc)` — each call gets a different timestamp, making exact reproducibility of graph snapshots impossible.

## 7. Recommended Fixes

### Priority 1 — Data Correctness (fix immediately)

| Fix | File | Change |
|-----|------|--------|
| **Fix fallback pair key** | `assay_merge.py:105-110` | Add 5th part to match parser: `"\|".join([pdb_id, chain_str, "none", "unknown", task_type])` |
| **Normalize BindingDB affinity scale** | `bindingdb.py:_parse_affinity()` | Convert nM → M before log10, or store as `-log10(M)`. Add unit field to schema. |
| **Guard log10(0)** | `bindingdb.py:_parse_affinity()` | Add `if std <= 0: return None` before `math.log10(std)` |
| **Fix metal disambiguation** | `rcsb_classify.py:~596` | Check per-metal proximity to small molecules instead of blanket reclassification |

### Priority 2 — Silent Failure Prevention

### Summary of Critical Findings

- **2 confirmed high-severity bugs** (fallback pair key format, affinity scale mixing) that produce silently wrong data
- **2 confirmed medium-severity bugs** (metal misclassification, dict mutation) that affect classification accuracy
- **8 silent failure paths** where errors are swallowed and partial/wrong data propagated
- **7 pipeline breakpoints** that can halt or corrupt the workflow
- **5 data integrity risks** including train/test leakage through graph features

The most urgent fix is the pair key format mismatch (BUG-1) — it silently breaks deduplication and assay linkage for any entry hitting the fallback path. Second priority is the affinity scale normalization (BUG-3) to prevent mixed-scale ML targets.
